What it does

Everything the grow room needs, on the device

A real-time control loop, an embedded single-page dashboard and over-the-air updates — no Raspberry Pi, no external server.

🌡️

VPD-based control

Temperature and humidity are fused into VPD and regulated with asymmetric hysteresis and per-stage day/night targets.

🌱

Grow-stage profiles

Seedling → Vegetative → Bloom → Drying, each with its own targets, photoperiod and irrigation — auto-advancing on an NTP clock.

💧

Precision irrigation

Pulse/soak watering with substrate profiles (soil, coco, perlite) and dry-back targets, logged to flash.

📊

Live web dashboard

Single-page UI served straight from the ESP32, with WebSocket telemetry every 2 s and a zoomable history chart.

🔌

8-channel relays

Fans, humidifier, dehumidifier, heat mat, lights, watering and A/C — each with minimum on/off timers and safety interlocks.

🛡️

Resilient by design

Dual-core FreeRTOS, non-blocking 100 ms loop, hardware watchdog, NVS-persisted settings and brown-out-safe scheduling.

📡

Remote-friendly

Over-the-air firmware updates, optional DuckDNS dynamic DNS and HTTP Basic Auth for access from outside the LAN.

🔒

No cloud, no account

Everything runs on the device. Your data never leaves your network — fully self-hosted.

🌬️

A/C integration

Climate interlocks and adjustable thresholds keep temperature in range while coordinating humidity and airflow.

🏠

Scale to many rooms

Run one ESP32 per grow room — each with its own stage, targets and schedule — then watch them side by side in one split-screen dashboard.

Scalable system

One controller per room. One dashboard for all.

Each grow room runs its own ESP32 with independent stages, VPD targets and schedules. Flip on split view to monitor a second room right beside the first — no extra app, no cloud.

Under the hood

Two cores, zero blocking

Core 0 runs the deterministic control loop; Core 1 handles WiFi, the web server and storage — so the UI never stalls the relays.

Get started

Flash it in minutes

Built with PlatformIO. Add your WiFi and credentials to a gitignored secrets.h, then flash over USB or the air.

# 1 · clone
git clone https://github.com/bn-jo/VPDcontrol.git && cd VPDcontrol

# 2 · add your secrets (kept out of the repo)
cp src/secrets.h.example src/secrets.h
#   edit src/secrets.h → WiFi SSID/password, web-auth + OTA password

# 3 · flash over USB (first time)
pio run -e esp32dev --target upload

# 4 · thereafter, update wirelessly
pio run -e ota --target upload

# open http://<device-ip>/ — the dashboard is served from the ESP32
